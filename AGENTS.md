# AGENTS.md

## Project Editing Rules
- Prefer editing files inside `content/` for portfolio updates.
- Do not change layout, styling, or component structure unless the user explicitly asks for layout changes.
- Keep the current visual design intact when making content-only updates.
- When adding or updating portfolio information, update the relevant page folder in `content/` first and only touch components if the schema changes.

## Primary Content Sources
- `content/shared/site.js`
- `content/home/`
- `content/about/`
- `content/tech/`
- `content/timeline/`
- `content/work/`
- `content/work/projects/`

## Commands
- Install: `npm install`
- Dev: `npm run dev`
- Build: `npm run build`

## Goal
- This portfolio should be easy to update by changing structured content files grouped by page and by project.
